##### Running Containers

`docker <object-type> <command> <options>`
	`object-type`  can be `container`, `image`, `network`, or a `volume` object.
	`command` indicates the action i.e. `run`
	`options` can be any valid parameter

To start a container following this syntax, we can do:
`docker container run <image name>`

**Publishing Ports**

To allow access outside of a container, you must publish the appropriate port inside the container to a port on your local network.

The common syntax for the `--publish` or `-p` option is as follows:
`--publish <host port>:<container port>`

**Other commands**

| Command / Argument                      | Action                                  |
| --------------------------------------- | --------------------------------------- |
| `docker container ls`                   | List containers (modern alternative)    |
| `docker container ps`                   | List containers                         |
| `--all`, `-a`                           | List all containers (even stopped ones) |
| `--detach`, `-d`                        | Run in the background                   |
| `--name`                                | Specify a name                          |
| `docker container rename <old> <new>`   | Rename a container                      |
| `docker container stop <identifier>`    | Stop a container (SIGTERM)              |
| `docker container kill <identifier>`    | Stop a container (SIGKILL)              |
| `docker container start <identifier>`   | Restart a container that exists         |
| `docker container restart <identifier>` | Reboot a running container              |
| `docker container create <image>`       | Creates a container but doesn't run     |
| `docker container rm <identifier`       | Remove dangling container               |
| `docker container prune`                | Remove all dangling containers          |
| `--rm`                                  | Remove after running                    |
| `-it`                                   | Access interactive shell when available |
| `-i`, `--interactive`                   | Access input stream of the container    |
| `-t`, `--tty`                           | Formats like a terminal, pseudo-tty     |

Every container has two identifiers: a name (random two words generated by docker) and a container ID (random 64-character long string).
